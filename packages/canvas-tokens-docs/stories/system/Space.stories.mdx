# System Space Tokens

System space tokens create consistent layout spacing across our UI. In v4, space tokens have been reorganized into semantic `padding`, `gap`, and `size` tokens to provide clearer intent and better alignment with common CSS properties.

## Usage

Space tokens can be consumed as a JavaScript / TypeScript object. Each token value references the CSS variable from a base token, so you'll need to have those CSS variables imported in your application as well.

```ts
// styles.ts
import {system} from '@workday/canvas-tokens-web';

const styles = {
  display: 'grid',
  gap: `var(${system.gap.md})`,
  padding: `var(${system.padding.md})`,
  height: `var(${system.size.lg})`,
};
```

They can also be consumed as CSS, Sass, or Less variables. In the example below, we're applying the same styles using CSS variables directly.

```css
// styles.css
.card {
  display: grid;
  gap: var(--cnvs-sys-gap-md);
  padding: var(--cnvs-sys-padding-md);
  height: var(--cnvs-sys-size-lg);
}
```

## Size Tokens

Size tokens provide semantic size values for height and width of elements. These tokens are used for component dimensions, not spacing.

## Padding Tokens

Padding tokens are spacing values that represent the internal spacing within a component, from its content to external boundaries.

padding tokens map directly to the padding CSS properties and its variants. These include but are not limited to padding, padding-inline-start, padding-inline-end, padding-top, padding-bottom, padding-left, padding-right.

## Gap Tokens

Gap tokens are spacing values that represent a distance between two elements. For example, the space between two buttons, icon to text, between two sections, or pills. Gap tokens may be applied inline or to vertical stacks interchangeably.
