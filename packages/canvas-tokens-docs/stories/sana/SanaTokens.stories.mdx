# Sana Theme Tokens

The Sana token set is an alternate theme for Canvas tokens. The values are scoped to a
`[data-theme="sana"]` selector, so they only apply when an ancestor element opts in to the theme.
This makes it possible to render the Sana palette inside an otherwise default Canvas application.

## Usage

### CSS

Import the Sana stylesheet alongside the standard token stylesheets. Then add `data-theme="sana"` to
any element you'd like to opt in (commonly the `<html>` or `<body>` element, or a single subtree if
you want to scope the theme).

```css
/* styles.css */
@import '@workday/canvas-tokens-web/css/base/_variables.css';
@import '@workday/canvas-tokens-web/css/brand/_variables.css';
@import '@workday/canvas-tokens-web/css/system/_variables.css';
@import '@workday/canvas-tokens-web/css/component/_variables.css';
@import '@workday/canvas-tokens-web/css/sana/_variables.css';
```

```html
<!-- Apply the theme to a whole document... -->
<html data-theme="sana">
  ...
</html>

<!-- ...or just a subtree. -->
<section data-theme="sana">
  <button class="cnvs-button">I use Sana tokens</button>
</section>
```

### JavaScript / TypeScript

Each token level (`base`, `brand`, `system`) ships a `sana` entry point. Unlike the standard JS
exports, every value is pre-wrapped with `var()` and includes the Sana fallback inline, so it can be
assigned directly to a style property.

```ts
// styles.ts
import '@workday/canvas-tokens-web/css/sana/_variables.css';
import {base, brand, system} from '@workday/canvas-tokens-web';

const styles = {
  backgroundColor: system.sana.color.bg.default,
  color: system.sana.color.fg.default,
  borderColor: brand.sana.neutral200,
  outlineColor: base.sana.blueA300,
};
```

> The Sana CSS variables only resolve to their themed values when `data-theme="sana"` is present on
> an ancestor element. If the attribute is missing, the fallback baked into each `var()` expression
> is used instead.
